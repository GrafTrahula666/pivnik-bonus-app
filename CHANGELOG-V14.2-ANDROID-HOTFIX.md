# V14.2 — исправление Android-загрузки

Исправлено:

- Telegram SDK теперь загружается до основного `app.js`, без гонки `async`/`defer`.
- Если SDK Android WebView не успел инициализироваться, `initData` извлекается напрямую из параметра запуска `tgWebAppData` в URL.
- Кнопка «Открыть облегчённо» больше не перезагружает страницу по кругу: она включает облегчённый режим и повторяет вход внутри текущего окна.
- Таймер аварийной загрузки больше не затирает настоящую причину ошибки.
- При отсутствии данных Telegram показывается диагностический код `A14-TG0`.
- В облегчённом режиме полностью отключены тяжёлые изображения и эффекты загрузочного экрана.
- Версии клиентских файлов подняты до `14.2`, чтобы Android WebView не использовал старый кэш.

## Проверка

1. Загрузить проект в Railway вместо V14.
2. Дождаться успешного деплоя.
3. Полностью закрыть Mini App на Android и открыть заново кнопкой из бота.
4. При первом тесте не использовать старую вкладку Telegram, оставшуюся открытой до деплоя.
5. Если появится ошибка, прислать её текст целиком. Код `A14-TG0` означает, что приложение открыли не как Mini App из кнопки бота либо Telegram не передал параметры запуска.
